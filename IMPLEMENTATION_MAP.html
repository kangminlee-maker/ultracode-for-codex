Current Architecture

AreaCurrent contract
Hybrid skillskills/ultracode-for-codex/SKILL.md positions native Codex Ultra for ad-hoc model-directed delegation and this package for durable workflows. The main Codex context owns decisions, edits, phase synthesis, progress projection, and critical re-checks; the CLI owns schema-enforced, journaled fan-out and resume anchors of jobId/runId/cwd.
CLI skillskills/ultracode-for-codex-cli/SKILL.md owns explicit CLI runtime operations: background jobs, attached runs, packaging, release checks, runtime-boundary validation, installed E2E, and local runtime artifacts.
CLIsrc/cli.ts exposes durable run, validate, resume, background control, skill sync, and setup surfaces. Before creating runtime truth, an attached child prepares the Codex backend: model/list resolves the effective model and validates the run-level effort. setup version 2 reports that same selected model and Ultracode-safe effort set. Progress remains stderr JSONL and the total result/failure channel remains stdout JSON.
Settingssettings.json owns workflow execution, progress, permission, retry, timeout, heartbeat, background paths, and the default run-level effort. The current default remains xhigh for compatibility; explicit medium, high, and catalog-supported max are first-class paths.
Runtimesrc/runtime/workflow-runtime.ts owns parsing, validation, scheduling, permission, retry, cancel, resume, and progress. An agent without explicit effort inherits the run-level effort. Built-in task is read-only analysis: its planner is medium and workers inherit the run tier. code-review level high is medium/high; the default deep profile is high/xhigh. max is accepted, while ultra is structurally rejected.
Journalsrc/runtime/workflow-journal.ts owns durable hash-chained workflow truth.
Model catalogsrc/codex/model-catalog.ts parses paginated app-server catalog responses, selects explicit/inherited/default models, exposes only runtime-safe efforts, and fails unsupported model/effort combinations before a turn.
Subagent backendsrc/codex/subagent-backend.ts has no hard-coded model fallback. It prepares the live catalog before the runtime journals an agent-capable run, strips provider credentials, serves bounded workspace tools, and caps Codex multi-agent V2 at one total thread so workers cannot create unjournaled descendants. Workers are read-only unless a permission-reviewed custom script requests worktree isolation. On POSIX, app-server shutdown terminates only the backend-owned process group, including shell wrappers; Windows currently terminates the direct child.
PackagePackage contents are the CLI runtime, install guide, both Codex skill command folders, docs, and public type exports. CLI --version and Codex clientInfo.version read package metadata, so packaging and npm version bumps update runtime surfaces together.

Current Decisions And Risks

Decision or riskCurrent state
Native versus durable orchestrationNative Ultra is the lightweight route; Ultracode is selected when schema, journal, cache, background, or reproducibility guarantees matter.
Effort boundaryPackaged default is medium (lowered from xhigh 2026-07-14 after the review-quality A/B); medium/high are the balanced Sol path; max stays single-agent; ultra cannot enter a workflow agent request.
Implementation ownershipThe built-in task analyzes but does not edit. Main-context edits are canonical; custom worktree writes remain isolated and require explicit integration.
Residual quality riskClosed for review workloads: the two-arm live effort A/B (2026-07-14) shows review quality saturates at medium (100% detection, 0 FP, judge quality 3/3 across medium/high/xhigh) on both single-module and hard cross-file tasks; only latency scales with effort. See docs/20260714-effort-quality-ab.md.
