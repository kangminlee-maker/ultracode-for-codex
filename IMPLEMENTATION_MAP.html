Current goalReleased 0.6.0 (npm latest). The native-Workflow parity roadmap is complete: contract-correct pipeline()/parallel(), nested workflow() (full-scope names + concurrent), edit-and-iterate resume, and an opt-in capability stream giving subagents web search, worktree file writes, MCP servers, and native per-agent types. Main riskThe capability stream widened the worker's authority surface: web/MCP are egress channels, file-write runs unsandboxed with a path guard as sole containment, and nested full-scope exec is gated only by the permission record. All are opt-in/default-off; that gate is the mitigation. Windows terminates only the direct child.
Recent decisionConcurrent nested workflow() (2026-07-16): the VM value projector moved from a shared ctx slot to a per-execution scope, so concurrent children marshal into their own realms and the sequential guard is removed. Child return projected on success; errors propagate raw (no double-projection).

Current Architecture

AreaCurrent contract
Hybrid skillskills/ultracode-for-codex/SKILL.md positions native Codex Ultra for ad-hoc model-directed delegation and this package for durable workflows. The main Codex context owns decisions, edits, phase synthesis, progress projection, and critical re-checks; the CLI owns schema-enforced, journaled fan-out and resume anchors of jobId/runId/cwd.
CLI skillskills/ultracode-for-codex-cli/SKILL.md owns explicit CLI runtime operations: background jobs, attached runs, packaging, release checks, runtime-boundary validation, installed E2E, and local runtime artifacts.
CLIsrc/cli.ts exposes durable run, validate, resume, background control, skill sync, and setup surfaces. Before creating runtime truth, an attached child prepares the Codex backend: model/list resolves the effective model and validates the run-level effort. setup version 2 reports that same selected model and Ultracode-safe effort set. Progress remains stderr JSONL and the total result/failure channel remains stdout JSON.
Settingssettings.json owns workflow execution, progress, permission, retry, timeout, heartbeat, background paths, and the default run-level effort. The current default remains xhigh for compatibility; explicit medium, high, and catalog-supported max are first-class paths.
Runtimesrc/runtime/workflow-runtime.ts owns parsing, validation, scheduling, permission, retry, cancel, resume, and progress. An agent without explicit effort inherits the run-level effort. Built-in task is read-only analysis: its planner is medium and workers inherit the run tier. code-review level high is medium/high; the default deep profile is high/xhigh. max is accepted, while ultra is structurally rejected.
Journalsrc/runtime/workflow-journal.ts owns durable hash-chained workflow truth.
Model catalogsrc/codex/model-catalog.ts parses paginated app-server catalog responses, selects explicit/inherited/default models, exposes only runtime-safe efforts, and fails unsupported model/effort combinations before a turn.
Subagent backendsrc/codex/subagent-backend.ts has no hard-coded model fallback. It prepares the live catalog before the runtime journals an agent-capable run, strips provider credentials, serves bounded workspace tools, and caps Codex multi-agent V2 at one total thread so workers cannot create unjournaled descendants. Workers are read-only by default; the capability gates below add tools per run inside the same turn (accounting/journal intact). It provisions an isolated Codex home per run (web_search config, allowlisted [mcp_servers.*] slices, per-agent persona via thread/start.developerInstructions). On POSIX, shutdown terminates only the backend-owned process group; Windows terminates the direct child.
Orchestration primitivesagent(), parallel()/pipeline() (native-contract: stage gets (prev, item, index), 4096-item cap), and nested workflow(). A per-run concurrency pool bounds dispatch (--agent-concurrency); a per-run output-token ceiling self-paces (--budget). All spend is one hash-chained journal; resume replays the unchanged prefix.
Capability gates (opt-in, default-off)Per-run flags that widen the worker's tool surface inside the Codex turn: --agent-web-search (native web_search), --agent-file-write (worktree-confined write_file/str_replace; the runtime path guard is the sole containment), --agent-mcp <s1,s2> (named-allowlist MCP servers sliced verbatim from ~/.codex/config.toml + auto-approved), and --agent-types (per-agent model + effort + persona from ~/.codex/agents/*.toml). Off (the default) is byte-identical; agentType is per-agent and enters the call key (resume auto-restores it).
Nested workflow()A child runs on the parent ctx (shared pool/budget/abort/journal chain); it emits only agent.*, keeping the journal single-run. Gated by --nested-workflows. Full-scope names (project/user/plugin) are permission-record-gated (mid-run can't prompt → fail-loud if unapproved). The value projector is per-execution, so children may run concurrently. One-level only; {scriptPath} deferred.
PackagePackage contents are the CLI runtime, install guide, both Codex skill command folders, docs, and public type exports. CLI --version and Codex clientInfo.version read package metadata, so packaging and npm version bumps update runtime surfaces together.

Current Decisions And Risks

Decision or riskCurrent state
Capability stream = worker authorityThe in-turn tool classes (web/file-write/MCP/agent-types) keep provider-reported turn accounting intact, so they were the safe subset to add; fan-out escapes (shell, multi-agent delegation) stay off. Each capability is opt-in, default-off, byte-identical when unused — that gate is the mitigation for the widened egress/exfil and unsandboxed-write surface. ultra effort stays structurally rejected (it would escape journal/cache/cost accounting), enforced even for agentType-supplied effort.
Nested-child authorityA mid-run nested workflow() cannot show a permission prompt, so a permission-required child (project/user/plugin) runs only against an existing allow record — else fail-loud. A child can never exceed the parent's approved isolation review (no silent widening). {scriptPath} nesting deferred for lack of a clean gate.
Realm marshaling under concurrencyConcurrent nested children each own a per-execution projector scope; the child return projects on success while errors propagate raw (double-projecting an error strips it to [object Object]). The hash chain is safe — the call-key critical section is synchronous, the same concurrency top-level parallel() already had.
Effort boundaryPackaged default is medium (lowered from xhigh after the 2026-07-14 review-quality A/B; quality saturates at medium, only latency scales). max stays single-agent; ultra cannot enter a workflow agent request.
Verification methodEvery capability/parity PR ran design-doc → GATE-0 real-path probe → adversarial design-verify (findings re-verified vs code) → default-off impl → falsifiability → dogfood/bot gate. Residual: no independent human review to date (self-merged after CI + design-verify); the async PR bot is inconsistent; the built-in code-review dogfood is blocked by a pre-existing Scope-phase gap.
