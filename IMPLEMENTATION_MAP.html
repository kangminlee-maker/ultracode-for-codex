Current Architecture

AreaCurrent contract
Hybrid skillskills/ultracode-for-codex/SKILL.md is the default user-facing command. The Codex main context owns adaptive phase planning, phase synthesis, visual progress snapshots from references/progress-visuals.md, and the final critical re-check recommendation; fan-out phases execute through the CLI runtime (validate-then-launch background runs, schema-enforced outputs, journaled results, resume anchors of jobId/runId/cwd). Codex-native subagents remain the fallback when the CLI is unavailable.
CLI skillskills/ultracode-for-codex-cli/SKILL.md owns explicit CLI runtime operations: background jobs, attached runs, packaging, release checks, runtime-boundary validation, installed E2E, and local runtime artifacts.
CLIsrc/cli.ts exposes run (including --validate for no-token authoring checks and --resume-from-run-id for all terminal source classes), background status/wait/logs/result/cancel/jobs/archive (status/jobs project runId and cwd), help, install guide, and package-version reporting. settings.json defaults to OS background execution so long Codex-launched runs can continue while Codex performs other work; --execution attached keeps progress on stderr JSONL and result on stdout JSON when blocking is desired. Completed runs add a phase/agent angle summary and a critical re-check recommendation for the current session LLM.
Settingssettings.json owns workflow execution mode, progress mode, permission default, retry limit, opt-in timeout, background file names, and Codex model controls. The packaged timeout default is 0, so workflows wait for completion, cancellation, or app-server exit.
Runtimesrc/runtime/workflow-runtime.ts owns parsing, validation, phase-wise parallel scheduling, permission, retry, cancel, resume, and progress events. Script agent() calls accept per-agent effort/model options with byte-compatible default cache identity; duplicate logical keys fail at reservation. Resume accepts completed, failed, cancelled, and interrupted sources (journal-first discovery, widened exact-key reuse, model adoption, workspace-drift disclosure). Built-in code-review runs finder-class agents at high and verdict-class agents at xhigh. Runtime-owned workflow state and isolated worktrees are preserved for review.
Journalsrc/runtime/workflow-journal.ts owns durable hash-chained workflow truth.
Subagent backendsrc/codex/subagent-backend.ts talks to Codex app-server over stdio, strips direct provider credentials, runs turns in the requested workflow cwd, and serves bounded read-only workspace tools.
PackagePackage contents are the CLI runtime, install guide, both Codex skill command folders, docs, and public type exports. CLI --version and Codex clientInfo.version read package metadata, so packaging and npm version bumps update runtime surfaces together.
